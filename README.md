# AI Tutor

> Upload any document. Pick your topic. Learn through an interactive, voice-driven AI tutoring session.

AI Tutor is a full-stack application that turns any PDF into a personalised, conversational learning experience. A Retrieval-Augmented Generation (RAG) pipeline powered by Google Gemini reads your document, delivers an engaging spoken lesson, and answers your follow-up questions in real time — all inside your browser.

---

## Who Is This For?

AI Tutor is built around a single idea: **any document can become a lesson**. That makes it useful across a wide range of industries and roles:

### 🎓 Education & E-Learning Platforms
Universities, bootcamps, and online course providers can embed AI Tutor to let students upload lecture notes, textbooks, or research papers and immediately receive a guided, voice-narrated explanation — reducing the time instructors spend re-explaining foundational material.

### 🏢 Corporate Training & Onboarding
HR and L&D teams can package policy handbooks, compliance documents, or product manuals as PDFs and have new hires learn through interactive sessions instead of passive reading. Employees can ask follow-up questions and get instant answers grounded in the actual document.

### ⚖️ Legal & Compliance
Law firms and compliance departments can upload contracts, regulations, or internal procedures. Attorneys and analysts can quiz the document in plain English, dramatically cutting the time spent hunting for specific clauses or understanding dense legal language.

### 🏥 Healthcare & Medical Education
Medical students and healthcare professionals can upload clinical guidelines, drug reference sheets, or research papers and receive a spoken summary with the ability to drill deeper on any point — ideal for exam prep or keeping up with fast-moving literature.

### 🔬 Research & Knowledge Management
Individual researchers and knowledge workers can use AI Tutor as a private reading assistant. Upload a paper or report, have it explained out loud, and interrogate it interactively — turning passive reading into an active conversation with the material.

### 🛍️ Sales & Product Enablement
Sales teams can upload product spec sheets, competitor analyses, or pricing guides and spin up a self-serve learning session before a customer call — without waiting for a trainer or digging through long documents manually.

## Features

| Feature | Description |
|---|---|
| **PDF Upload** | Drop any PDF (up to 10 MB) and the backend extracts, chunks, and embeds it into a FAISS vector store |
| **RAG-Powered Teaching** | Google Gemini generates an 8–10 sentence teaching script grounded in your document's content |
| **Voice Playback (TTS)** | The lesson is read aloud sentence-by-sentence via the Web Speech Synthesis API with mute/unmute control |
| **Raise Your Hand** | Interrupt the tutor at any time to ask a question — type it or speak it with voice recognition |
| **Voice Input (STT)** | Web Speech Recognition API transcribes your spoken question directly into the input field |
| **Contextual Q&A** | Follow-up answers are generated by the same RAG pipeline scoped to the uploaded document |
| **Session Management** | Sessions are stored in Redis with a 1-hour TTL; each IP is limited to one free session |
| **Per-session limits** | Up to 20 questions per session; question rate-limited to 1 per 30 seconds to prevent API abuse |
| **Landing Page** | Animated React landing page with feature highlights and a call-to-action |

## Architecture

```
┌─────────────────────────────────────────────────┐
│                  Browser (React)                 │
│                                                  │
│  Landing Page → Setup (upload PDF + topic)       │
│       ↓                                          │
│  Session Page                                    │
│   • polls /session/:id/next for sentences        │
│   • speaks each sentence via Web Speech TTS      │
│   • accepts typed / spoken questions             │
│   • posts questions to /session/:id/question     │
└──────────────────────┬──────────────────────────┘
                       │  HTTP (JSON / multipart)
┌──────────────────────▼──────────────────────────┐
│               Flask API  (port 7700)             │
│                                                  │
│  POST /upload                                    │
│   1. Extract text from PDF (pdfplumber)          │
│   2. Chunk → embed → store in FAISS              │
│   3. RAG query → teaching script                 │
│   4. Push sentences to Redis queue               │
│                                                  │
│  GET  /session/:id/next  → pop next sentence     │
│  POST /session/:id/question → RAG answer         │
└──────────────────────┬──────────────────────────┘
         ┌─────────────┴──────────────┐
         ▼                            ▼
   Redis (queues,              FAISS index
   metadata, rate limits)      (disk, per session)
         │
         └──── Google Gemini Embeddings + LLM
```

## Tech Stack

### Backend (`b-ai-tutor/server`)
- **Python / Flask** — REST API
- **pdfplumber** — PDF text extraction
- **LangChain** — orchestration, text splitting, RAG chain
- **FAISS** — local vector similarity search
- **Google Gemini** — `gemini-embedding-001` for embeddings, `gemini-2.5-flash` for generation
- **Redis** — sentence queues, session metadata, rate-limiting
- **flask-limiter** — per-route and per-IP rate limits
- **flask-cors** — configurable CORS origins

### Frontend (`f-ai-tutor`)
- **React 18 + TypeScript + Vite**
- **Tailwind CSS + shadcn/ui** — design system
- **Web Speech API** — Text-to-Speech (TTS) and Speech Recognition (STT)
- **React Router v6** — client-side routing

## Getting Started

### Prerequisites
- **Python 3.11+**
- **Node.js 18+ / Bun**
- **Redis** running locally (default: `localhost:6379`)
- A **Google AI API key** with access to the Gemini API

### 1. Clone the repo

```bash
git clone https://github.com/adityaraj61522/ai-tutor.git
cd ai-tutor
```

### 2. Backend setup

```bash
cd b-ai-tutor/server

# Create and activate a virtual environment
python3 -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Create a .env file
cp .env.example .env   # or create manually — see below
```

**`.env` variables:**

```env
GOOGLE_API_KEY=your_google_api_key_here

# Redis connection (defaults shown)
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=0
REDIS_PASSWORD=

# Server settings
PORT=7700
FLASK_DEBUG=0

# Set to "1" only when deployed behind a reverse proxy (e.g. nginx)
TRUST_PROXY=0

# Comma-separated allowed frontend origins
CORS_ORIGINS=http://localhost:8080
```

**Start the server:**

```bash
python app.py
# or
bash run_server.sh
```

The API will be available at `http://localhost:7700`.

### 3. Frontend setup

```bash
cd f-ai-tutor

# Install dependencies
bun install   # or: npm install

# Start the dev server
bun run dev   # or: npm run dev
```

Open `http://localhost:8080` in your browser.

## API Reference

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/` | Health check |
| `POST` | `/upload` | Upload PDF + topic; returns `session_id` |
| `GET` | `/session/:id/next` | Pop next sentence from the teaching queue |
| `POST` | `/session/:id/question` | Submit a follow-up question; queues answer sentences |

### `POST /upload`

**Form fields:**  `uploadedPDF` (file), `topicToLearn` (string, max 300 chars)

**Response:**
```json
{ "session_id": "uuid-v4", "sentence_count": 9 }
```

**Limits:** One successful upload per IP address.

### `POST /session/:id/question`

**Body:** `{ "question": "..." }` (max 500 chars)

**Response:**
```json
{ "status": "queued", "sentence_count": 5 }
```

**Limits:** 1 request per 30 seconds, max 20 questions per session.

## Security & Rate Limiting

- **IP-based one-time upload limit** — each IP may create exactly one session; enforced atomically in Redis (`SET NX`)
- **Per-route rate limits** via `flask-limiter` backed by Redis
- **Input validation** — UUID format check on session IDs; length caps on all text fields
- **CORS** — restricted to the origins listed in `CORS_ORIGINS`
- **File size cap** — 10 MB enforced both client-side and server-side
- **Text length cap** — extracted text truncated to 50,000 chars before embedding to control API costs
